# 内容管理系统（CMS）使用指南

本网站内置了一个隐藏的内容管理系统，方便你管理和更新网站内容。

## 🔑 如何打开CMS

有两种方式打开管理面板：

### 方法1：快捷键（推荐）
在任何页面按下：`Ctrl + Shift + M`

### 方法2：隐藏触发器
点击页面右下角的隐藏区域（10x10像素的透明区域）

## 📋 CMS功能

### 1. 个人信息管理
- 编辑姓名、职称、单位
- 更新邮箱和联系方式
- 修改个人简介和介绍

### 2. 研究方向管理
- 添加新的研究方向
- 编辑现有研究方向
- 删除研究方向
- 为每个方向添加学生和研究内容

### 3. 研究成果管理
- **期刊论文**：添加/编辑论文信息
- **会议论文**：管理会议发表
- **专利**：记录专利信息
- **科研项目**：管理项目列表
- **获奖情况**：添加获奖记录

### 4. 教学工作管理
- **授课课程**：添加/编辑课程信息
- **编写教材**：记录教材出版
- **教学获奖**：管理教学奖项

### 5. 学生指导管理
- 在读学生（博士/硕士/本科）
- 已毕业学生

### 6. 竞赛指导管理
- 添加竞赛获奖记录
- 编辑竞赛信息
- 上传竞赛照片

## 🚀 导出功能

### 1. 时间线简历
生成按时间顺序排列的学术简历，包含所有教育背景、工作经历、研究成果等。

支持格式：
- **Markdown**（.md）：适合GitHub等平台
- **HTML**：可在浏览器中打印为PDF
- **PDF**：需要先导出HTML再打印

使用方法：
1. 打开CMS（Ctrl+Shift+M）
2. 点击"导出功能"标签
3. 在"时间线简历"部分选择格式
4. 点击相应按钮下载

### 2. 研究成果列表
导出所有研究成果的完整列表，包括论文、专利、项目、获奖等。

支持格式：
- **Markdown**（.md）：结构化文本格式
- **HTML**：网页格式
- **BibTeX**（.bib）：学术引用格式，可直接导入LaTeX

### 3. 成果统计报告
生成各类成果的统计报告和可视化图表。

### 4. 完整数据备份
导出网站所有数据的JSON格式备份文件（siteData.json）。

## 💾 数据管理

### 保存数据
1. 在CMS中修改数据后
2. 点击底部的"💾 保存更改"按钮
3. 数据会保存到浏览器本地存储（localStorage）

### 下载数据备份
1. 点击"📥 下载数据"按钮
2. 保存JSON文件到本地
3. 妥善保管此文件（包含所有网站数据）

### 上传数据
1. 点击"📤 上传数据"按钮
2. 选择之前下载的JSON文件
3. 数据将被导入并覆盖当前数据

## 📝 数据文件位置

所有网站数据存储在：`assets/js/data.js`

你可以直接编辑此文件来批量更新数据。

## 🔒 安全性说明

1. **隐藏性**：CMS的打开方式对普通访客是隐藏的
2. **本地存储**：数据保存在浏览器本地，不会上传到服务器
3. **访问控制**：只有知道快捷键或触发位置的人才能打开CMS
4. **数据备份**：定期下载数据备份以防丢失

## ⚠️ 注意事项

1. **浏览器兼容性**：
   - Chrome/Edge（推荐）
   - Firefox
   - Safari

2. **数据持久化**：
   - 浏览器本地存储的数据可能被清除（清除浏览数据时）
   - 务必定期下载备份

3. **更新网站**：
   - 修改数据后，需要刷新页面查看效果
   - 某些修改可能需要重新生成静态页面

4. **文件上传**：
   - 图片需要手动上传到对应文件夹
   - CMS只管理数据，不处理文件上传

## 📚 快速开始

### 第一次使用

1. **打开CMS**
   ```
   按 Ctrl + Shift + M
   ```

2. **编辑个人信息**
   - 点击"个人信息"标签
   - 填写你的真实信息
   - 点击"保存更改"

3. **添加研究成果**
   - 点击"研究成果"标签
   - 选择成果类型（期刊/会议/专利等）
   - 点击"+ 添加"按钮
   - 填写表单
   - 保存

4. **下载备份**
   - 点击"导出功能"标签
   - 点击"下载数据备份"
   - 保存JSON文件

### 日常更新流程

1. 打开CMS（Ctrl+Shift+M）
2. 选择要更新的内容类别
3. 添加/编辑/删除条目
4. 保存更改
5. 下载最新备份
6. 关闭CMS

## 🎯 最佳实践

1. **定期备份**：每次重大更新后下载数据备份
2. **版本控制**：将data.js文件加入Git版本控制
3. **数据一致性**：确保手动编辑的数据格式正确
4. **测试功能**：重要更新后在不同浏览器测试
5. **图片管理**：按照assets/images/README.md的说明组织图片

## 🛠️ 高级功能

### 批量编辑数据
直接编辑`assets/js/data.js`文件：

```javascript
const siteData = {
    personal: {
        name: "你的姓名",
        email: "your-email@example.com",
        // ...
    },
    // ...
};
```

### 自定义导出格式
修改`assets/js/cms.js`中的导出函数：
- `generateTimelineCVMarkdown()`
- `generateAchievementsListMarkdown()`
- `generateBibTeX()`

## 📞 问题反馈

如遇到问题：
1. 检查浏览器控制台（F12）的错误信息
2. 确认数据格式是否正确
3. 尝试清除浏览器缓存后重试
4. 恢复之前的数据备份
